## Rollback, Idempotency, Reconciliation

ROLLBACK_PLAN=READY_OWNER_GATED
IDEMPOTENCY_PLAN=READY
RECONCILIATION_PLAN=READY

Rollback must never delete pre-existing data. It may only act on artifacts proven by pilot ID, correlation ID, and fingerprint to be created by the pilot. Audit evidence must be preserved.

The second execution with the same selected message must produce no duplicate Drive file, no duplicate Sheets row, no duplicate active job, and deterministic source identity evidence.

The reconciliation table for D6J-B must include Gmail message ID, attachment ID, source fingerprint, SHA-256, Firestore job ID, Firestore attachment ID, Drive file ID, Drive metadata identity, Sheets business key, Sheets row identity, audit correlation ID, and final job status. Raw live IDs belong in a local ignored manifest only, not in committed docs.
