## Access Model Draft

```text
CLIENT_FRONTEND_READ_ONLY=YES
ANONYMOUS_ACCESS=NO
DIRECT_CLIENT_WRITES=NO
GAS_SERVICE_IDENTITY_FUTURE_WRITES_ONLY=YES
OWNER_ADMIN_FINDING_ACCESS=YES
```

Future frontend users may read only sanitized projections/findings appropriate to their role. Direct client writes to invoice jobs, commit plans, audit events, reports, or repair controls are disallowed.

## Collection Design Draft

```text
invoiceJobs=SERVICE_WRITE_OWNER_ADMIN_READ
invoiceJobs/{jobId}/events=APPEND_ONLY_SERVICE_WRITE
invoiceJobs/{jobId}/reconciliationReports=SERVICE_WRITE_OWNER_ADMIN_READ

```

Security invariants:

```text
APPEND_ONLY_EVENTS=YES
IMMUTABLE_COMMIT_PLAN=YES
NO_RAW_GMAIL_ID=YES
NO_RAW_DRIVE_ID=YES
NO_EMAIL_BODY=YES
NO_XML_CONTENT=YES
NO_PDF_CONTENT=YES

SAFE_FIELD_ALLOWLIST=YES
```

## Retention Draft

```text
JOB_RETENTION=ACCORDING_TO_AUDIT_POLICY
EVENT_RETENTION=BOUNDED_OR_ARCHIVED_POLICY
REPORT_RETENTION=LATEST_PLUS_AUDIT_REQUIRED_HISTORY
TTL_CANDIDATE=SHORT_LIVED_DISCOVERY_ONLY
MAXIMUM_DOCUMENT_SIZE=FUTURE_FIRESTORE_LIMIT_GUARD
MAXIMUM_EVENT_COUNT_PER_JOB=BOUNDED
RAW_SOURCE_PAYLOAD_RETENTION=NEVER
```

The future production design should archive or compact old events without allowing event mutation, retain enough reports for audit/reconciliation, and never persist raw Gmail bodies, XML/PDF contents, or raw source IDs.

No deployable rules are created in D5E. GAS does not yet have production Firestore credentials/runtime.
