## Collection Schema

```text
COLLECTION_SCHEMA=invoiceJobs/{jobId};invoiceJobs/{jobId}/events/{eventId};invoiceJobs/{jobId}/reconciliationReports/{reportId}
OPTIONAL_PROJECTION_COLLECTION=invoiceJobProjections/{jobId}
XML_PAYLOAD_COLLECTION=FORBIDDEN
PDF_PAYLOAD_COLLECTION=FORBIDDEN
EMAIL_BODY_COLLECTION=FORBIDDEN
```

`invoiceJobs/{jobId}` is the canonical durable shadow state document. Events and reconciliation reports are append-only subcollections.

## Job Document Contract

```text
IMMUTABLE_COMMIT_PLAN_RULE=YES
APPEND_ONLY_EVENTS=YES
REPORT_ONLY_RECONCILIATION=YES
EXECUTION_MODE=SHADOW
PRODUCTION_MUTATION_ALLOWED=false
SHADOW_STATUS_ALLOWLIST=SHADOW_DETECTED;SHADOW_EVALUATING;SHADOW_READY;SHADOW_REVIEW_REQUIRED;SHADOW_CONFLICT;SHADOW_FAILED
PRODUCTION_COMPLETION_STATES_FORBIDDEN=FILES_SAVED;COMMITTING;ROWS_COMMITTED;PROJECTIONS_COMMITTED;COMPLETED
```

Immutable after creation or first save:

```text
IMMUTABLE_FIELDS=jobId;createdAt;invoiceIdentityHash;commitPlan;commitPlanHash;executionMode;productionMutationAllowed
VERSION_INCREMENT=EXACTLY_ONE_PER_ACCEPTED_MUTATION
EXPECTED_VERSION_CONFLICT_POLICY=REJECT_AND_SURFACE
```

Allowed mutable shadow fields:

```text
MUTABLE_FIELDS=status;shadowEvaluationStatus;version;updatedAt;lastObservedAt;latestReconciliationStatus;latestFindingCodes;safe provenance hashes
```

## Rules

```text
RULES_FILE=firestore.rules
CLIENT_WRITE_POLICY=DENY
ANONYMOUS_ACCESS_POLICY=DENY
FRONTEND_READ_POLICY=DENY_BY_DEFAULT
NORMAL_AUTHENTICATED_CLIENT_WRITE=DENY
ADMIN_READ=DISABLED_UNLESS_FUTURE_CLAIM_CONTRACT_EXISTS
SERVICE_WRITE_RULES_MODEL=NOT_REPRESENTED_BY_CLIENT_RULES
ADMIN_SDK_RULES_BYPASS_ACKNOWLEDGED=YES
BACKEND_ALLOWLIST_ENFORCEMENT_REQUIRED=YES
```

Security rules are a client access boundary. A future service identity that uses an Admin SDK or equivalent server credential bypasses Firestore Security Rules, so the backend adapter must still enforce allowlists, immutability, size limits, and fail-closed sanitization.
